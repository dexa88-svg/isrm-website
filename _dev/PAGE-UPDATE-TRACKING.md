# Page Update Tracking & Last Updated Tags

**Version:** 1.0  
**Effective Date:** 2026-05-08  
**Purpose:** Ensure all published pages display a "Last Updated" date so users and maintainers can track content freshness

---

## Overview

Every publicly visible page on ISRM must display a **"Last Updated"** date/timestamp. This allows:
- Users to know how current the content is
- Maintainers to identify stale content requiring refresh
- The automated content sync task to verify which pages have been recently validated
- Easy sorting by "most recently updated" content

---

## Requirement: Mandatory for All Pages

### Pages That MUST Include "Last Updated"
✅ All repair guides (HTML in `/public/repair-guides/`)  
✅ All diagnostic guides (HTML in `/public/diagnostics/`)  
✅ All model overview pages (HTML in `/public/models/`)  
✅ All video guide pages (HTML in `/public/videos/`)  
✅ News and community pages  
✅ Parts catalog pages  

### Pages Where "Last Updated" is Optional
- Homepage (`index.html`) — shows instead as "Site Last Updated"
- Index/listing pages (repair-guides/index.html, etc.) — optional, but recommended

---

## Display Format

### Location on Page

**Option A: In Meta Section (Preferred for Repair Guides)**
Display immediately below the title, in the metadata bar alongside "Added" date:

```html
<div class="meta">
  Added: 2026-05-07 &nbsp;|&nbsp; 
  <strong>Last Updated: 2026-05-08</strong>
  &nbsp;|&nbsp; 
  Sources: <a href="...">Source 1</a>, <a href="...">Source 2</a>
  &nbsp;|&nbsp; 
  <span class="tag">GY6</span> <span class="tag">Repair</span>
</div>
```

**Option B: In Footer (For Reference Pages)**
Display in the page footer with other metadata:

```html
<footer class="guide-footer">
  <p>Last Updated: <strong>2026-05-08</strong></p>
  <p>Content reviewed and validated by ISRM community</p>
  <p><a href="../index.html">ISRM Home</a></p>
</footer>
```

**Option C: Both Meta + Footer (Maximum Visibility)**
Show in both locations for important guides that receive frequent updates.

## Date Format Requirements

| Context | Format | Example |
|---------|--------|---------|
| HTML meta section | YYYY-MM-DD | 2026-05-08 |
| Footer text | Long format | May 8, 2026 |
| Page attributes | ISO 8601 | 2026-05-08T14:30:00Z |
| Filename metadata | YYYYMMDD | 20260508 |

**Standard:** Use `YYYY-MM-DD` (2026-05-08) as the primary format for consistency.

## Update Procedure

### When to Update the "Last Updated" Date

**Update EVERY TIME you:**
- ✏️ Fix typos, grammar, or language clarity
- 🔄 Revise step-by-step instructions
- 🎥 Add, replace, or update embedded videos
- 📊 Update specifications, part numbers, or compatibility tables
- 🔗 Add or fix links and citations
- 🎨 Modify styling or formatting
- 🆔 Change the meta description or title
- ✅ Verify accuracy against source material

**Do NOT update if:**
- Only updating internal navigation links (global changes)
- Only fixing HTML structure (no content changes)
- Only fixing CSS variables (appearance-only fixes)

### How to Update

**Step 1:** Before saving any changes, note today's date: **2026-05-08**

**Step 2:** Locate the `<div class="meta">` section in the guide:
```html
<div class="meta">
  Added: 2026-05-07 &nbsp;|&nbsp; 
  <strong>Last Updated: 2026-05-07</strong>
  ...
</div>
```

**Step 3:** Change the "Last Updated" date to today:
```html
<div class="meta">
  Added: 2026-05-07 &nbsp;|&nbsp; 
  <strong>Last Updated: 2026-05-08</strong>
  ...
</div>
```

**Step 4:** If using footer metadata, update there too:
```html
<footer class="guide-footer">
  <p>Page auto-generated by ISRM Content Sync on 2026-05-08. 
  Last Updated: 2026-05-08.</p>
  ...
</footer>
```

**Step 5:** Save and commit with a message that references the update:
```bash
git commit -m "Update GY6 carburetor guide: clarify fuel screw positioning (last updated: 2026-05-08)"
```

## Automated Update Tracking

### Content Sync Task

The **`isrm-content-sync`** scheduled task runs daily and:
- Scans all guide pages for "Last Updated" dates
- Compares against known sources (forums, YouTube, blogs)
- Flags guides that haven't been updated in 30+ days
- Generates a report: `_dev/documentation/CONTENT_AGE_REPORT.md`

### Consistency Check Task

The **`isrm-consistency-check`** task runs daily and:
- Verifies every public page has a "Last Updated" date
- Flags pages missing this metadata
- Suggests updates for obvious errors or outdated info

### Design Compliance Task

The **`isrm-design-compliance`** task runs Mondays at 9:00 AM and:
- Validates that "Last Updated" dates match the expected format
- Checks that dates are recent (not older than 6 months)
- Generates compliance report

## Style Guide: CSS for "Last Updated"

The `.meta` class already handles styling. If you need to emphasize the updated date:

```css
.meta strong {
  color: var(--accent);  /* Orange highlight */
  font-weight: 600;
}

.meta strong::before {
  content: "✓ ";  /* Optional: adds a checkmark */
}
```

Result: `✓ Last Updated: 2026-05-08` (highlighted in orange)

## Monitoring & Audits

### Weekly Content Age Audit

Every Monday, the system generates `CONTENT_AGE_AUDIT_[DATE].md` showing:
- Pages updated in last 7 days (✅ Fresh)
- Pages updated 7–30 days ago (⚠️ Recent)
- Pages updated 30–90 days ago (⚠️ Stale)
- Pages updated 90+ days ago (❌ Very Stale)

### Manual Content Review

Review schedule (quarterly):
- Verify all "Last Updated" dates are accurate
- Remove outdated information (videos gone, links broken)
- Refresh content based on new sources or community feedback
- Update the date when validated

## Troubleshooting

### Problem: "What if I make a tiny fix?"
**Answer:** Update the date anyway. Even typo fixes matter for users wanting the latest version.

### Problem: "What if I'm fixing a date/link error from yesterday?"
**Answer:** Update to today's date. You're correcting and validating the content.

### Problem: "Multiple people edit the same guide. Whose date counts?"
**Answer:** Update the date each time. The final "Last Updated" is when the last person finished editing.

### Problem: "Should I add an 'Updated by' field?"
**Answer:** Optional. You can add this in a future version, but focus on the date first. Example:
```html
<div class="meta">
  Last Updated: 2026-05-08 by @Den
  ...
</div>
```
